# Data Scientist Resume Matcher with Explainable NLP Ranking

This project is an NLP-based resume screening and comparison tool designed specifically for **Data Scientist** roles. It reads a job description and a dataset of resumes, then compares candidates using three different matching strategies: a **basic keyword matcher**, a **hard-skill-focused matcher**, and a more advanced **recruiter-oriented concept matcher**. The goal is to show how simple text overlap can be improved by using role-specific logic that better reflects how recruiters and hiring managers evaluate resumes.

The project was built to move beyond generic keyword counting. A simple keyword checker often gives high importance to broad or less meaningful terms, while this approach separates out **technical hard skills** such as Python, SQL, machine learning, statistics, and cloud tools, and also identifies higher-level **job-relevant concepts** such as data analysis, feature engineering, stakeholder communication, experimentation, MLOps, and business problem solving. This makes the scoring more interpretable and more aligned with real hiring expectations for data science positions.

Each resume is scored in three ways. First, the project calculates a **basic keyword score** based on direct overlap between cleaned job-description terms and resume terms. Second, it calculates a **hard-skill-only score** based on matches to predefined Data Scientist technical skills. Third, it calculates a **complex weighted score** that combines hard-skill matches and recruiter-relevant concept matches into a single ranking metric. This allows direct comparison between a naive baseline and a more domain-aware matching strategy.

The project also includes multiple visualizations to make the results explainable and portfolio-friendly. These include score distribution comparisons, top matched hard skills, top matched concepts, category-level score analysis, ranking-change comparisons between methods, heatmaps by candidate category, and an explainability chart showing which matched features contributed to the top candidate’s score. These visuals help demonstrate not only the final ranking, but also *why* certain candidates rank higher under different matching approaches.

Overall, this project demonstrates practical skills in **text preprocessing, rule-based NLP, feature engineering, scoring design, visualization, and explainable candidate ranking**. It is intended as both a resume screening prototype and a portfolio project showing how domain knowledge can improve basic NLP matching for recruitment analytics.

The resume dataset used in this project can be found on Kaggle here: https://www.kaggle.com/datasets/snehaanbhawal/resume-dataset
